# MIRAGEA AI 生态讲解说明

这份文档给路演讲解人使用，目标是讲清楚两件事：

- MIRAGEA AI 哪里用了 HTX / 火币生态。
- MIRAGEA AI 哪里用了 B.AI。

不要把它讲成“我们贴了几个生态 Logo”，而要讲成一个完整产品闭环：

```text
B.AI 生成内容和估值
用户用 HTX / U / 法币参与互动和交易
IP 币承载每部剧的市场价值
创作者获得分润和增长反馈
```

## 1. 一句话版本

MIRAGEA AI 是一个 AI 短剧共创和 IP 资产市场。B.AI 负责生成剧情分支、辅助创作者生产内容，并根据预告片、剧情介绍和观众热度生成 IP 币估值；HTX / 火币生态负责支付、资产展示、市场交易和创作者分润。

## 2. HTX / 火币生态怎么讲

### 2.1 HTX 是支付入口

讲解重点：

用户不只是看短剧，还可以参与剧情投票、预测市场和 IP 币交易。HTX 在这里作为重要支付方式之一。

页面演示：

1. 进入 `发现`。
2. 打开剧情投票。
3. 展示高权重投票里的 `HTX 支付`。
4. 投票完成后，点击 `查看该剧市场`。
5. 打开任意剧集详情。
6. 点击买入，展示支付方式里有 `HTX`。

讲解话术：

```text
这里我们把 HTX 放进用户参与剧情共创和市场交易的支付链路里。用户可以用 HTX 做高权重投票，也可以在市场里用 HTX 参与预测和 IP 币相关交易。
```

### 2.2 HTX 是资产和分润口径

讲解重点：

MIRAGEA AI 有用户资产、市场收益、创作者分润，所以需要一个资产和账本体系。Demo 中先用测试网模拟，生产阶段可以对接 HTX API 做资产、行情、充值提现和账本同步。

页面演示：

1. 进入 `我的`。
2. 展示 `HTX 资产总览`。
3. 进入 `消息`。
4. 展示资产到账、市场结算、创作者分润消息。

讲解话术：

```text
短剧不是一次性消费内容，而是一个持续共创和交易的生态。HTX 在这里承担资产展示、收益结算、创作者分润和未来账本同步的角色。
```

### 2.3 HTX 是市场交易的一部分

讲解重点：

市场页不只是普通预测市场，而是包含剧情预测、IP 币发行、交易热榜和剧集详情行情。HTX 是交易支付方式，也是动态流水中的交易资产。

页面演示：

1. 进入 `市场`。
2. 打开 `交易热榜`。
3. 展示 HTX 成交记录。
4. 打开剧集详情。
5. 展示 IP 币价格走势、交易数据、资金流和持有人数据。

讲解话术：

```text
市场里既有剧情预测，也有每部剧自己的 IP 币。HTX 既可以作为支付资产，也可以出现在交易流水和创作者收益里。
```

## 3. B.AI 怎么讲

### 3.1 B.AI 帮创作者生成内容

讲解重点：

创作者不需要从零写所有剧情分支。B.AI 可以根据当前短剧设定生成下一集分支、冲突方向和可供用户投票的选项。

页面演示：

1. 进入 `我的`。
2. 找到 `创作者工具`。
3. 点击 `新建剧集`。
4. 展示 B.AI 生成的剧情分支和估值结果。

讲解话术：

```text
创作者工具里接入了 B.AI。创作者可以用它快速生成剧情分支，这些分支会进入用户投票和后续市场预测，形成内容生产和用户共创的闭环。
```

### 3.2 B.AI 帮 IP 币做估值

讲解重点：

IP 币不是凭空发行。B.AI 会结合预告片、剧情介绍、题材潜力、用户互动热度、投票热度和交易热度，生成一个估值参考。

页面演示：

1. 进入 `市场`。
2. 切到 `IP 币发行`。
3. 点击某部剧的 `B.AI 估值`。
4. 页面显示 `生成中`。
5. 生成完成后，卡片上的热度和风险分数会更新。

讲解话术：

```text
IP 币发行前，B.AI 会先做内容和市场信号分析。它不是直接给投资建议，而是帮助平台和创作者判断这部剧当前的热度、风险和适合的发行估值区间。
```

### 3.3 B.AI 在剧详情页解释价格

讲解重点：

用户进入某部剧详情页后，不只看到价格，还能看到 B.AI 为什么给出这个估值。这样 IP 币不是只有 K 线，而是和剧情内容、观众反馈联系起来。

页面演示：

1. 在 `市场` 打开任意剧集详情。
2. 找到 `IP 币详情`。
3. 展示价格走势、交易数据、资金流、持有人数。
4. 展示 `B.AI 估值` 卡片。
5. 点击 `刷新估值`，展示新的分支建议和估值摘要。

讲解话术：

```text
传统行情页只告诉用户价格涨跌。MIRAGEA AI 会进一步解释价格背后的内容原因：剧情潜力、观众热度、风险评分和下一步分支建议都由 B.AI 生成。
```

## 4. 推荐演示顺序

### 第一步：先讲短剧共创

打开 `发现`：

- 展示用户看短剧。
- 展示剧情投票。
- 展示 HTX 高权重投票。
- 投票封存后点击 `查看该剧市场`。

核心表达：

```text
用户不只是观众，而是剧情共同创作者。
```

### 第二步：再讲市场

打开 `市场`：

- 展示剧情预测。
- 展示 IP 币发行。
- 展示交易热榜。
- 展示 HTX 支付和交易数据。

核心表达：

```text
用户参与剧情后，剧情热度会进一步进入市场，形成 IP 资产价值。
```

### 第三步：最后讲 B.AI

打开 `我的` 和 `市场详情`：

- 在创作者工具里展示 B.AI 生成剧情分支。
- 在 IP 币发行里展示 B.AI 生成估值。
- 在剧详情页展示 B.AI 解释估值原因。

核心表达：

```text
B.AI 不是孤立按钮，而是连接内容生产、用户反馈和 IP 币发行的引擎。
```

## 5. 软件整体逻辑

当前产品逻辑可以这样讲：

```text
看剧
→ 投票决定剧情
→ B.AI 生成下一步剧情分支
→ 市场根据投票生成预测盘口
→ 热门剧集进入 IP 币发行
→ B.AI 根据内容和热度辅助估值
→ 用户用法币 / U / HTX / IP 币参与支付和交易
→ 创作者获得分润和数据反馈
→ 下一集继续生成
```

这个逻辑是顺的，因为它把短剧内容、用户共创、AI 生产、市场交易和创作者收益串成了一个循环。

## 6. 目前 Demo 的边界

讲解时要说清楚：

- 当前是测试网模拟盘。
- HTX、USDT、预测代币、IP 币都是 Demo 资产。
- 不涉及真实充值、提现、现金交易或投资建议。
- B.AI 当前通过统一接口接入，Demo 阶段可先用 GLM 模型跑真实 AI 生成；未来拿到 B.AI 官方 Key 后可替换为 B.AI。

推荐话术：

```text
为了保证 Demo 稳定，我们把资产交易做成测试网模拟，把 B.AI 做成可替换的 AI Provider。现在可以先用 GLM 跑真实 AI 生成，未来接入主办方 B.AI 时不需要改产品流程。
```

## 7. 最短版讲解稿

```text
MIRAGEA AI 把 AI 短剧做成一个可共创、可交易、可持续生成的 IP 市场。

HTX / 火币生态主要承担支付、资产、市场交易和创作者分润。用户可以用 HTX 做高权重投票，也可以在市场交易和 IP 币认购中选择 HTX 支付。

B.AI 主要承担内容生成和估值。创作者用 B.AI 生成剧情分支；市场用 B.AI 根据预告片、剧情介绍、观众热度和交易数据生成 IP 币估值；剧详情页再用 B.AI 解释价格背后的剧情潜力和风险。

所以这个产品不是简单短剧播放器，而是“AI 生成内容 + 用户共创 + HTX 支付资产 + IP 币市场”的闭环。
```
