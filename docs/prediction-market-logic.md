# MIRAGEA AI 剧情预测市场逻辑

## 1. 核心定义

Miragea 在每集结束后开启剧情走向投票，同时基于本轮投票选项生成预测盘口。

盘口标的不是剧情本身，而是：

```text
本集投票截止时，哪个剧情选项获得最高总权重
```

盘口类型包括：

- CP 走向
- 关键反转
- 结局走向

每集默认生成 3-5 个可预测选项。

## 2. 生命周期

单盘口生命周期：

```text
24-72 小时
```

流程：

1. 投票窗口开启前，系统根据创作者预设分支自动生成盘口。
2. 投票窗口开启时，预测盘口同步开盘。
3. 投票窗口关闭时，预测盘口同步关闭。
4. 后台生成投票截止快照。
5. 剧情更新后，基于快照结果执行结算。

## 3. 做市机制

早期采用 LMSR 自动做市。

目的：

- 解决短周期、多盘口场景的冷启动问题
- 无需对手盘也可成交
- 给每个剧情选项提供连续报价
- 避免新剧集、新分支没有流动性

随着头部剧集交易深度提升，可演进为：

```text
LMSR AMM + CLOB 订单簿 + 专业做市商
```

## 4. 平台收入

Take Rate：

```text
2%
```

Demo 中只展示手续费逻辑，不执行真实扣费或真实分润。

## 5. 清算依据

清算依据为投票截止快照。

比赛 Demo 口径：

```text
测试网模拟盘：前端展示链上快照与结算状态，但不执行真实资金结算。
```

生产版口径：

```text
投票截止后生成可审计快照，剧情更新后由结算合约或合规交易服务完成清算。
```

## 6. 支付通道

产品展示支付通道：

- USDT
- HTX
- 信用卡
- PayPal
- Web3 钱包

生产版可评估：

- HTX API 资产与流水同步
- 支付服务商通道
- 钱包链上结算
- 法币入金合规链路

比赛 Demo 顶层标记为测试网模拟盘；内部界面按未来商业化产品展示，便于后续商业化迭代时减少改动。
