# MIRAGEA AI 市场逻辑

## 1. 核心定义

原“预测市场”统一改名为“市场”。市场包含两类核心能力：

- 剧情预测：每集结束后开启剧情走向投票，并基于本轮投票选项生成预测盘口。
- IP 币：为单部剧、系列剧或角色 IP 提供发行、交易、行情和估值展示。

创作者工具入口不放在市场一级导航，统一放在“我的”页面，作为创作者个人工作台入口。

盘口标的不是剧情本身，而是：

```text
本集投票截止时，哪个剧情选项获得最高总权重
```

盘口类型包括：

- CP 走向
- 关键反转
- 结局走向

每集默认生成 3-5 个可预测选项。

## 2. 市场信息架构

底部导航展示为：

```text
市场
```

市场首页模块：

- 剧情预测
- IP 币发行
- 新币热榜
- 交易热榜
- B.AI 估值榜

剧详情页模块：

- 剧情简介、预告片、角色信息
- 当前投票和预测盘口
- IP 币详情
- 价格走势、交易数据、持仓分布
- B.AI 估值卡片

我的页面模块：

- 创作者工具
- 资产
- 订单
- 分润
- 收藏

## 3. 剧情预测生命周期

单盘口生命周期：

```text
24-72 小时
```

流程：

1. 投票窗口开启前，系统根据创作者预设分支自动生成盘口。
2. 投票窗口开启时，预测盘口同步开盘。
3. 投票窗口关闭时，预测盘口同步关闭。
4. 后台生成投票截止快照。
5. 剧情更新后，基于快照结果执行结算。

## 4. 做市机制

早期采用 LMSR 自动做市。

目的：

- 解决短周期、多盘口场景的冷启动问题
- 无需对手盘也可成交
- 给每个剧情选项提供连续报价
- 避免新剧集、新分支没有流动性

随着头部剧集交易深度提升，可演进为：

```text
LMSR AMM + CLOB 订单簿 + 专业做市商
```

## 5. IP 币发行规则

IP 币发行平台放在“市场”内，不单独做一级导航。

发行对象：

- 单部剧 IP
- 系列剧 IP
- 角色 IP
- 番外或衍生内容 IP

发行规则需要展示：

- 发行主体：创作者、平台联名或项目方
- 总供应量
- 初始发行价
- 初始流通量
- 创作者份额、社区份额、流动性份额、平台储备
- 锁仓和解锁计划
- 认购方式：白名单、公开认购、空投、流动性池
- 使用场景：剧情投票加权、提前看、会员权益、周边、番外解锁、打赏
- 风险提示：价格波动、流动性、项目延期、内容审核、地区合规

比赛 Demo 口径：

```text
IP 币为测试网模拟资产，仅用于展示发行、交易、估值和资产详情流程，不具备现金价值。
```

## 6. B.AI 估值方式

B.AI 需要结合内容侧和市场侧数据，为 IP 币提供估值参考。估值结果用于发行定价辅助、风险展示和市场热度解释，不作为投资建议。

输入数据：

- 预告片：题材、角色、视觉质量、冲突密度、可传播片段
- 剧情介绍：世界观、人物关系、反转点、分支延展性
- 观众热度：播放、完播、点赞、评论、收藏、分享
- 共创热度：投票人数、加权投票金额、分支讨论量
- 交易热度：预约认购、成交量、持有人数、价格波动

B.AI 输出：

- 热度评分
- 剧情潜力评分
- 社区增长评分
- 流动性评分
- 风险评分
- 建议发行价区间
- 建议初始流通量
- 估值理由摘要

## 7. 剧详情页 IP 币详情

每部剧详情页都展示该剧 IP 币详情。信息密度可以参考 GMGN 的资产行情页：让用户能快速看价格、成交、资金流和持仓变化，但视觉语言保持 MIRAGEA 自有设计。

建议展示：

- 当前价格
- 24h 涨跌幅
- 市值 / FDV
- 流通量 / 总供应量
- 持有人数
- K 线或价格走势
- 24h 交易量
- 买卖笔数
- 资金流入 / 流出
- 最新成交列表
- 大户或聪明钱动态
- 持仓分布
- 发行规则
- 解锁计划
- B.AI 估值卡片
- 风险提示

## 8. 平台收入

Take Rate：

```text
2%
```

Demo 中只展示手续费逻辑，不执行真实扣费或真实分润。

收入来源包括：

- 剧情预测交易手续费
- IP 币发行服务费
- IP 币交易手续费
- 高权重投票支付
- 会员权益和内容解锁
- 创作者工具增值服务

## 9. 清算依据

清算依据为投票截止快照。

比赛 Demo 口径：

```text
测试网模拟盘：前端展示链上快照与结算状态，但不执行真实资金结算。
```

生产版口径：

```text
投票截止后生成可审计快照，剧情更新后由结算合约或合规交易服务完成清算。
```

## 10. 支付通道

产品展示支付通道：

- 法币：信用卡、PayPal 或本地支付服务商
- U：USDT / U 本位余额
- HTX：HTX Pay / HTX 余额
- IP 币：指定剧集 IP 币
- Web3 钱包

需要明确露出 HTX 支付的场景：

- 预测交易
- IP 币认购
- 高权重投票
- 会员权益
- 创作者工具增值服务

生产版可评估：

- HTX API 资产与流水同步
- 支付服务商通道
- 钱包链上结算
- 法币入金合规链路
- IP 币余额支付与兑换路径

比赛 Demo 顶层标记为测试网模拟盘；内部界面按未来商业化产品展示，便于后续商业化迭代时减少改动。
