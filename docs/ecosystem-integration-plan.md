# MIRAGEA AI 主办方生态融合方案

项目要求：建议使用至少 1 项主办方生态方向，包括 HTX 相关 API、$HTX 经济模型、B.AI 算力。

MIRAGEA AI 建议三项都融合，但优先级不同。

## 1. 最推荐：$HTX 经济模型

这是和产品最自然的一项，建议作为核心叙事。

可直接融合到：

- 高权重剧情投票
- 剧情预测盘口交易手续费
- 创作者分润
- 贡献者奖励
- 会员权益
- 手续费折扣
- 平台治理或头部剧集投票权重

产品表达：

```text
$HTX 是 Miragea AI 内部共创、预测、分润和权益体系的核心资产。
用户可使用 $HTX 购买高权重投票，参与市场中的剧情预测与 IP 币交易；创作者可获得 $HTX 分润；长期贡献用户可获得手续费折扣与治理权益。
```

当前 Demo 已体现：

- Market 资产看板中展示 `$HTX`
- Weighted Vote 使用 `HTX PAY`
- 市场交易与支付抽屉中明确展示 HTX 支付入口
- 交易流水中展示 Creator Dividend
- Market 机制中展示 Take Rate 2%

## 2. 推荐：B.AI 算力

B.AI 和“AI 生长式短剧”强相关，建议作为技术亮点。

可直接融合到：

- 依据胜出投票结果生成下一集
- 为 3-5 个剧情分支生成预览图、分镜或短片段
- 自动生成剧集封面
- 根据预告片、剧情介绍、评论热度、投票热度、收藏与交易数据生成 IP 币估值参考
- 评论与剧情内容安全审核辅助
- 创作者后台的剧情分支扩写

产品表达：

```text
Miragea AI 使用 B.AI 算力在每轮投票结算后生成下一集内容，并为未胜出的分支保留预览资产；同时结合预告片、剧情介绍与观众热度生成 IP 币估值参考，形成内容生产、观众反馈、资产定价联动的 AI 短剧市场。
```

当前 Demo 已体现：

- 演示周期 `NEXT` 阶段为 AI 生成下一集
- Market 页生态接入中展示 B.AI Compute
- 市场页可新增 B.AI Valuation 模块，展示热度评分、剧情潜力评分与估值区间

建议下一步补：

- 创作者后台增加 `Generate with B.AI`
- 生成状态：Queued / Rendering / Review / Published
- 每个分支展示 B.AI 生成的预览封面
- IP 币发行页增加 `Estimate with B.AI`，根据预告片、剧情简介、互动数据生成发行建议

## 3. 可选但有用：HTX API

HTX API 更适合作为资产与行情底座，不建议在比赛 Demo 中真的接真实资金。

可直接融合到：

- `$HTX` 行情展示
- 用户资产同步
- 充值/提现状态查询
- 资产流水对账
- Market 价格、IP 币行情或交易相关数据聚合
- HTX 支付状态回调

产品表达：

```text
Miragea AI 预留 HTX API 作为资产、行情、充值提现与账本同步接口，比赛阶段以 mock API 展示完整交互，生产阶段接入真实 API 与合规支付链路。
```

当前 Demo 已体现：

- `/api/market` 返回 ecosystemStack
- Market 页面展示 HTX API 资产、行情、充值提现、账本同步能力
- 顶部钱包连接支持 Base Sepolia

## 4. 新产品要求：Market 与 IP 币

原“预测市场”统一改名为“市场”。市场不只承载剧情预测盘口，也承载每部剧的 IP 币发行、交易和资产数据。

### 信息架构

- 底部导航：`市场`
- 市场首页：剧情预测、IP 币发行、新币热榜、交易热榜
- 剧详情页：剧情信息、投票、预测盘口、IP 币行情、交易数据
- 我的页面：创作者工具入口、资产、订单、分润、收藏

创作者工具入口放在“我的”，作为个人工作台的一部分。入口名称建议为：

```text
Creator Studio / 创作者工具
```

### 支付方式

支付抽屉和交易确认页需要展示多种支付方式：

- 法币：信用卡、PayPal 或本地支付服务商
- U：USDT / U 本位余额
- HTX：HTX Pay / HTX 余额
- IP 币：使用指定剧集 IP 币支付会员权益、周边或二级内容

预测交易、IP 币认购、会员权益、高权重投票都要明确露出 HTX 支付选项。

### IP 币发行平台

IP 币发行入口放在“市场”内，不单独做一级导航。发行对象是单部剧或系列 IP。

发行规则建议包含：

- 发行主体：创作者、平台联名或项目方
- 绑定对象：单部剧、系列剧或角色 IP
- 总量与解锁：总供应量、创作者份额、社区份额、流动性份额、平台储备
- 初始发行方式：白名单认购、公开认购、空投、流动性池
- 使用场景：剧情投票加权、提前看、会员权益、周边、番外解锁、创作者打赏
- 风控规则：发行审核、锁仓期、交易冷却、异常交易监控、风险提示
- Demo 口径：比赛阶段全部为测试网模拟资产，无真实现金价值

### B.AI 估值方式

B.AI 不只用于生成内容，也用于 IP 币估值辅助。估值不是直接给投资建议，而是给发行定价和风险展示提供参考。

输入数据：

- 预告片内容、视觉风格、角色设定
- 剧情介绍、题材、世界观、分支可扩展性
- 观众热度：播放、完播、点赞、评论、收藏、分享
- 共创热度：投票人数、加权投票金额、分支讨论量
- 市场热度：认购预约、交易量、持有人数、价格波动

B.AI 输出：

- 热度评分
- 剧情潜力评分
- 社区增长评分
- 风险评分
- 建议发行价区间
- 建议流通量与解锁节奏
- 估值理由摘要

### 剧详情页 IP 币模块

每部剧详情页都要有该剧 IP 币详情，信息密度可以参考 GMGN 的行情页结构，但视觉上保持 MIRAGEA 自有风格。

建议展示：

- 当前价格、涨跌幅、市值、流通量、持有人数
- K 线或价格走势
- 交易量、买卖笔数、资金流入/流出
- 最新成交列表
- 大户或聪明钱动态
- 持仓分布
- 发行规则、解锁计划、风险提示
- B.AI 估值卡片：热度、剧情潜力、估值区间、风险说明

## 5. 当前 Demo 与生产版边界

当前 Demo 采用顶层模拟盘提示，但内部按未来商业化产品表达：

- 支付通道展示：法币 / U / HTX / IP 币 / 钱包
- 做市机制：LMSR
- Take Rate：2%
- 清算依据：投票截止链上快照
- 市场演进：LMSR AMM + CLOB + 专业做市商
- IP 币发行与交易：测试网模拟资产

生产版上线前需要补：

- 合规地区开关
- KYC / AML
- 真实支付服务商接入
- HTX API Key 管理与后端签名代理
- 资金账本、对账、风控
- 投票快照审计
- 市场反操纵机制
- IP 币发行审核与信息披露
- B.AI 估值模型审计与免责声明

