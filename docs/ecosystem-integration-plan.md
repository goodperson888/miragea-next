# MIRAGEA AI 主办方生态融合方案

项目要求：建议使用至少 1 项主办方生态方向，包括 HTX 相关 API、$HTX 经济模型、B.AI 算力。

MIRAGEA AI 建议三项都融合，但优先级不同。

## 1. 最推荐：$HTX 经济模型

这是和产品最自然的一项，建议作为核心叙事。

可直接融合到：

- 高权重剧情投票
- 剧情预测盘口交易手续费
- 创作者分润
- 贡献者奖励
- 会员权益
- 手续费折扣
- 平台治理或头部剧集投票权重

产品表达：

```text
$HTX 是 Miragea AI 内部共创、预测、分润和权益体系的核心资产。
用户可使用 $HTX 购买高权重投票，参与剧情预测市场；创作者可获得 $HTX 分润；长期贡献用户可获得手续费折扣与治理权益。
```

当前 Demo 已体现：

- Market 资产看板中展示 `$HTX`
- Weighted Vote 使用 `HTX PAY`
- 交易流水中展示 Creator Dividend
- Market 机制中展示 Take Rate 2%

## 2. 推荐：B.AI 算力

B.AI 和“AI 生长式短剧”强相关，建议作为技术亮点。

可直接融合到：

- 依据胜出投票结果生成下一集
- 为 3-5 个剧情分支生成预览图、分镜或短片段
- 自动生成剧集封面
- 评论与剧情内容安全审核辅助
- 创作者后台的剧情分支扩写

产品表达：

```text
Miragea AI 使用 B.AI 算力在每轮投票结算后生成下一集内容，并为未胜出的分支保留预览资产，形成可持续生长的 AI 短剧生产管线。
```

当前 Demo 已体现：

- 演示周期 `NEXT` 阶段为 AI 生成下一集
- Market 页生态接入中展示 B.AI Compute

建议下一步补：

- 创作者后台增加 `Generate with B.AI`
- 生成状态：Queued / Rendering / Review / Published
- 每个分支展示 B.AI 生成的预览封面

## 3. 可选但有用：HTX API

HTX API 更适合作为资产与行情底座，不建议在比赛 Demo 中真的接真实资金。

可直接融合到：

- `$HTX` 行情展示
- 用户资产同步
- 充值/提现状态查询
- 资产流水对账
- Market 价格或交易相关数据聚合

产品表达：

```text
Miragea AI 预留 HTX API 作为资产、行情、充值提现与账本同步接口，比赛阶段以 mock API 展示完整交互，生产阶段接入真实 API 与合规支付链路。
```

当前 Demo 已体现：

- `/api/market` 返回 ecosystemStack
- Market 页面展示 HTX API 资产、行情、充值提现、账本同步能力
- 顶部钱包连接支持 Base Sepolia

## 4. 当前 Demo 与生产版边界

当前 Demo 采用顶层模拟盘提示，但内部按未来商业化产品表达：

- 支付通道展示：USDT / HTX / 信用卡 / PayPal / 钱包
- 做市机制：LMSR
- Take Rate：2%
- 清算依据：投票截止链上快照
- 市场演进：LMSR AMM + CLOB + 专业做市商

生产版上线前需要补：

- 合规地区开关
- KYC / AML
- 真实支付服务商接入
- HTX API Key 管理与后端签名代理
- 资金账本、对账、风控
- 投票快照审计
- 市场反操纵机制

## 5. 比赛提交建议

如果只能重点讲 1 项：

```text
$HTX 经济模型
```

如果要讲技术壁垒：

```text
B.AI 算力 + 投票结算后自动生成下一集
```

如果要讲生态协同：

```text
HTX API + $HTX 经济模型 + B.AI 算力，构成支付/激励/内容生成闭环
```
