# MIRAGEA AI 路演宣讲文档

## 1. 项目一句话

MIRAGEA AI 是一个“AI 短剧共创 + 剧情预测市场 + IP 币行情”的互动内容平台。用户不只是观看短剧，而是可以参与剧情投票、预测后续走向、查看每部剧的 IP 币行情；创作者则通过 B.AI 提升内容生产效率，并通过 HTX 生态完成支付、资产展示、分润和交易闭环。

简短版：

> MIRAGEA AI 把短剧从“看完即走”的内容消费，升级成“用户共创、AI 生成、资产定价、创作者变现”的 IP 经济系统。

## 2. 产品解决什么问题

传统短剧平台主要有三个问题：

- 用户只是被动观看，互动深度低，热度很难持续沉淀。
- 创作者生产压力大，剧情分支、预告、封面和后续内容都依赖人工。
- 热门 IP 的价值无法被实时表达，平台、创作者和核心粉丝之间缺少可持续的收益闭环。

MIRAGEA AI 的解决方式是：

- 用剧情投票让用户参与共创。
- 用预测市场把剧情热度变成可观察的市场信号。
- 用 B.AI 生成剧情分支、创作建议和 IP 币估值参考。
- 用 HTX 生态承接支付、资产、交易、分润和用户激励。

## 3. 系统整体逻辑

完整闭环可以这样讲：

```text
用户看短剧
→ 对下一集剧情进行投票
→ 投票结果形成剧情走向和市场预测盘口
→ B.AI 根据剧情、预告片、评论、点赞、投票和交易热度生成分支建议
→ 热门剧集形成 IP 币行情页和估值参考
→ 用户用法币 / U / HTX / IP 币参与支付、预测和交易
→ 创作者获得分润、数据反馈和下一轮创作输入
→ 下一集继续生成，形成持续增长循环
```

这套逻辑的核心不是单点功能，而是一个循环：

```text
内容生产 → 用户互动 → 市场反馈 → AI 辅助创作 → 资产沉淀 → 创作者变现
```

## 4. HTX 生态融合

### 4.1 HTX 作为支付方式

在 MIRAGEA AI 里，用户可以在多个关键行为中看到 HTX：

- 高权重剧情投票。
- 预测市场下单。
- IP 币买卖模拟。
- 剧集解锁、会员权益或创作者支持。
- 钱包绑定和资产总览。

宣讲话术：

> HTX 在这里不是简单贴一个支付 Logo，而是进入了用户参与剧情、参与市场和支持创作者的核心链路。用户看剧、投票、交易和解锁权益时，都可以把 HTX 作为重要支付资产。

### 4.2 HTX 作为资产和账本口径

“我的”页面里展示 HTX 资产总览，预测资产和 IP 币资产也统一折算为 HTX 口径。这样讲解时更清楚：

- 用户知道自己平台资产总额。
- 创作者知道分润和收入。
- 平台可以统一展示交易流水、结算记录和收益通知。

生产版可以继续接入：

- HTX 行情。
- 资产同步。
- 充值提现状态。
- 账本对账。
- 支付状态回调。

宣讲话术：

> 对短剧平台来说，资产体系不是只服务交易，它还服务创作者分润、用户激励和平台账本。HTX 可以成为整个互动内容经济里的统一资产表达。

### 4.3 HTX 作为激励模型

HTX 可以延展为平台激励资产：

- 持有 HTX 获得更高剧情投票权重。
- 使用 HTX 支付获得手续费折扣。
- 创作者获得 HTX 分润。
- 高频互动用户获得 HTX 奖励或会员权益。
- 核心用户参与头部剧集治理或优先体验。

宣讲话术：

> MIRAGEA AI 希望把 HTX 从支付资产进一步延展成共创激励资产。用户越深度参与内容共创，越能获得权益；创作者越能制造热度，越能获得收益。

## 5. B.AI 融合

### 5.1 B.AI 辅助内容生产

B.AI 在创作者工具中承担内容生产助手角色：

- 根据当前剧情生成下一集分支。
- 给出可投票的剧情选项。
- 辅助生成预告片方向、标题、封面和剧情摘要。
- 根据用户反馈调整后续剧情重点。

宣讲话术：

> 创作者不需要从零构思所有分支。B.AI 可以把用户投票、评论和剧情设定转化成下一集创作建议，让短剧从线性生产变成 AI 辅助的互动生成。

### 5.2 B.AI 辅助 IP 币估值

B.AI 的估值不是投资建议，而是内容和市场信号分析。它会参考：

- 预告片吸引力。
- 剧情简介和题材潜力。
- 点赞、评论、收藏、分享。
- 投票人数和投票分布。
- 预测市场热度。
- IP 币价格、交易量、资金流和持有人变化。

输出内容包括：

- 估值区间。
- 预测周期，例如“未来 24 小时”。
- 热度评分。
- 剧情潜力评分。
- 风险评分。
- 估值摘要。

宣讲话术：

> 传统行情页只告诉用户价格涨跌。MIRAGEA AI 进一步用 B.AI 解释价格背后的内容原因：为什么这部剧热、为什么这个分支可能带来增长、风险在哪里，以及这个估值周期覆盖到什么时候。

### 5.3 B.AI 连接内容和市场

B.AI 的价值不只是生成几段文案，而是把三个系统连起来：

```text
剧情内容
用户反馈
市场信号
```

它让创作者知道该怎么继续写，让用户知道为什么这部剧有价值，也让平台知道哪些 IP 值得重点扶持。

## 6. 页面演示路线

### 第一步：发现页

讲解重点：

- 用户刷短剧。
- 生长剧可以参与剧情投票。
- 成品剧可以点赞、评论、收藏和切换集数。
- 剧集详情里可以查看 IP 币页签和预测市场页签。

推荐话术：

> 这里先让评委看到，MIRAGEA AI 不是传统播放器，而是一个可以边看边参与的短剧入口。

### 第二步：剧情投票

讲解重点：

- 用户可以选择剧情方向。
- 投票会影响后续剧情。
- 投票结果会进入预测市场。

推荐话术：

> 用户从观众变成共创者。每一次投票既影响剧情，也会形成后续市场数据。

### 第三步：市场

讲解重点：

- 市场里有剧情预测。
- 可以查看实时走势、持仓、评论、动态。
- 买入时展示多种支付方式：法币、U、HTX、IP 币。
- 规则文案只讲预测市场，不混入 IP 币发行解释。

推荐话术：

> 预测市场表达的是用户对剧情结果的判断。它把内容热度转化成可量化、可观察的市场反馈。

### 第四步：剧集 IP 币详情

讲解重点：

- 每部剧都有独立 IP 币行情。
- 展示价格走势、交易量、市值、持有人、流动性、资金流。
- B.AI 显示估值区间和预测周期。

推荐话术：

> 每部剧都可以形成自己的 IP 价值曲线。我们用类似行情页的方式展示交易数据，同时用 B.AI 解释这些价格背后的内容逻辑。

### 第五步：我的页面

讲解重点：

- 资产总览统一用 HTX 口径。
- 可以绑定钱包。
- 创作者工具入口在“我的”。
- 创作者可以用 B.AI 生成剧情分支和估值结果。

推荐话术：

> “我的”页面同时服务用户和创作者。用户看资产和记录，创作者用 B.AI 生成内容，并通过平台获得分润。

## 7. 变现逻辑

MIRAGEA AI 的商业模式可以分成五类。

### 7.1 内容消费收入

- 付费解锁剧集。
- 会员订阅。
- 提前看、番外篇、导演剪辑版。
- 用 HTX、U、法币或 IP 币支付。

### 7.2 预测市场手续费

- 用户参与剧情预测时，平台收取交易手续费。
- 高频互动剧集会带来持续交易量。
- 预测市场也能为平台提供内容热度数据。

### 7.3 IP 币交易和服务费

- 每部剧形成独立 IP 币行情。
- 平台可从交易、上架、流动性服务和数据服务中获得收入。
- B.AI 估值能力可以成为 IP 发行和运营的基础设施。

### 7.4 创作者工具收入

- B.AI 剧情生成。
- 分支扩写。
- 封面、预告、标题生成。
- 数据分析和爆款预测。
- 面向创作者的高级工具订阅。

### 7.5 品牌和广告收入

- 热门短剧植入。
- 剧情分支冠名。
- IP 联名活动。
- 粉丝任务和品牌互动。

一句话总结：

> 平台既赚内容消费的钱，也赚互动交易的钱，还能通过 B.AI 创作者工具和 IP 数据服务获得长期收入。

## 8. 为什么这个模式成立

### 8.1 用户侧

用户获得的不只是观看体验，而是参与感：

- 我投票决定剧情。
- 我预测剧情走向。
- 我持有或关注自己喜欢剧集的 IP 币。
- 我能通过评论、点赞和分享影响热度。

### 8.2 创作者侧

创作者获得更高效率和更清晰反馈：

- B.AI 提升创作速度。
- 投票结果告诉创作者用户想看什么。
- 市场数据告诉创作者哪个 IP 更有潜力。
- 分润机制让创作者从内容热度中持续受益。

### 8.3 平台侧

平台获得更强留存和更多收入层：

- 短剧内容带来流量。
- 投票和预测提高互动。
- IP 币行情沉淀资产关注。
- B.AI 降低内容生产成本。
- HTX 生态承接支付和资产闭环。

## 9. Demo 边界和合规口径

路演时建议主动说明：

- 当前是 Demo 和测试网模拟环境。
- 页面中的 HTX、U、IP 币和预测资产均为模拟资产。
- B.AI 估值是内容和市场信号分析，不构成投资建议。
- 生产版会接入 KYC、AML、地区限制、风控、账本审计和真实支付合规流程。
- Demo 阶段可以先使用 GLM 作为 AI Provider；未来接入 B.AI Key 后，产品流程不需要改变。

推荐话术：

> 为了保证演示稳定，我们把交易和资产做成模拟盘，把 AI 能力做成可替换 Provider。今天大家看到的是完整产品闭环，未来接入真实 B.AI 和 HTX API 后，核心路径可以平滑升级。

## 10. 路演结构建议

### 1 分钟版本

```text
MIRAGEA AI 是一个 AI 短剧共创和 IP 市场平台。用户可以看剧、投票决定剧情、参与剧情预测，并查看每部剧的 IP 币行情。HTX 生态负责支付、资产、交易和创作者分润；B.AI 负责剧情生成、创作者工具和 IP 币估值。我们把短剧从单纯内容消费，升级成用户共创、AI 生成和资产化运营的闭环。
```

### 3 分钟版本

```text
第一，用户在发现页看短剧，不只是观看，还能点赞、评论、投票决定剧情。

第二，投票结果进入预测市场，用户可以根据剧情走向进行预测，平台展示实时走势、成交记录和支付方式，其中 HTX 是核心支付资产之一。

第三，每部剧都有自己的 IP 币详情页，展示价格走势、交易量、资金流和持有人数据。B.AI 会根据剧情、预告片、互动热度和交易热度给出估值区间，并明确预测周期。

第四，创作者可以在“我的”页面使用 B.AI 生成剧情分支和内容建议，再根据用户反馈继续创作。创作者收益、用户资产和平台账本可以统一进入 HTX 资产体系。

所以 MIRAGEA AI 的价值是把内容、AI、用户互动和 Web3 资产连接成一个可持续增长的短剧 IP 经济系统。
```

## 11. 可以重点强调的亮点

- B.AI 不是孤立按钮，而是贯穿内容生成、估值解释和创作者工具。
- HTX 不是简单支付入口，而是支付、资产、激励、分润和账本体系。
- 预测市场只负责剧情结果判断，IP 币行情负责剧集资产价值展示，两者在页面上已经分开。
- 每部剧详情页都有“IP 币”和“预测市场”两个页签，逻辑更清楚。
- B.AI 估值显示预测周期，说明估值覆盖未来多久，表达更专业。
- 当前 Demo 是模拟资产和测试网口径，适合路演展示，也方便后续接真实生态能力。

```text
短剧行业已经证明了内容消费的规模，但下一步需要解决的是互动、留存和创作者收益。MIRAGEA AI 用 B.AI 提升内容生产效率，用 HTX 生态承接支付和资产流转，用剧情预测和 IP 币行情把用户热度沉淀成可观察的市场价值。

我们希望让每一部短剧都不只是内容，而是可以被共创、被定价、被持续运营的 AI 原生 IP。
```
