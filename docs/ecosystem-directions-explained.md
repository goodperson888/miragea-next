# 主办方生态方向快速理解与 MIRAGEA 融合方式

这份文档用最直白的方式解释：HTX API、$HTX 经济模型、B.AI 算力分别是什么、能做什么、为什么适合 MIRAGEA AI。

## 1. HTX 相关 API

### 是什么

HTX API 可以理解为交易所/资产平台提供给应用的接口能力。

它通常可用于：

- 查询资产余额
- 查询行情价格
- 查询充值/提现状态
- 查询订单或流水
- 做账本同步
- 做支付或资产相关状态回调

### 对 MIRAGEA 有什么用

MIRAGEA 有短剧付费、预测市场、创作者分润、用户资产看板，所以很需要资产与流水底座。

可以融入这些模块：

- Market 顶部资产看板
- 我的页面 HTX 资产总览
- 充值 / 提现状态
- 预测结算收益到账
- 创作者分润到账
- 交易流水与对账

### 当前 Demo 怎么体现

当前项目里已在 Market 页面展示：

```text
HTX API：资产、行情、充值提现、账本同步
```

并在 mock API 中预留：

```text
/api/market
```

后续真实接入时，前端不直接请求 HTX 私钥接口，应由后端做签名代理和风控。

### 一句话理解

```text
HTX API 是 MIRAGEA 的资产与账本同步底座。
```

## 2. $HTX 经济模型

### 是什么

$HTX 是 HTX 生态里的核心资产。放到 MIRAGEA 里，可以把它设计成用户、创作者、平台之间流转的经济媒介。

它不只是“支付币”，还可以承担：

- 权益
- 激励
- 分润
- 手续费折扣
- 治理权重
- 高权重投票

### 对 MIRAGEA 有什么用

这是最适合 MIRAGEA 的生态方向，因为你的产品本身有强经济循环：

1. 用户看剧。
2. 用户投票影响剧情。
3. 用户参与剧情预测。
4. 平台抽取 Take Rate。
5. 创作者获得分润。
6. 高贡献用户获得奖励或权益。

$HTX 可以融入：

- 付费解锁短剧
- 购买高权重投票
- 预测市场手续费
- 创作者链上分润
- 用户贡献榜奖励
- 会员等级
- 手续费折扣

### 当前 Demo 怎么体现

当前项目里已经有：

- 顶部 HTX 资产展示
- Weighted Vote 使用 HTX Pay
- Market 资产看板 `$HTX Balance`
- 创作者分润流水 `Creator Dividend`
- Take Rate 2%
- 我的页面 HTX 资产总览

### 一句话理解

```text
$HTX 是 MIRAGEA 的共创、预测、分润和会员权益核心资产。
```

## 3. B.AI 算力

### 是什么

B.AI 算力可以理解为主办方生态里的 AI 计算资源或 AI 技术组件。

它可以用于：

- 文生视频
- 图生视频
- 剧情分支生成
- 分镜生成
- 封面生成
- 内容审核辅助
- 评论审核辅助
- 创作者工作流加速

### 对 MIRAGEA 有什么用

MIRAGEA 的核心卖点是“生长式短剧”。生长式短剧不是一次性拍完，而是每一集根据用户投票继续生成。

B.AI 正好可以融入这个闭环：

1. 创作者设置 3-5 个剧情分支。
2. 用户投票。
3. 投票截止后产生胜出分支。
4. B.AI 根据胜出分支生成下一集。
5. 新一集上线后再次开启投票和预测盘口。

还可以用于：

- 给每个分支生成预览海报
- 给未胜出分支生成番外内容
- 给剧集超市生成封面
- 自动生成剧情摘要
- 审核违规内容

### 当前 Demo 怎么体现

当前项目里已经有：

- 演示周期 `NEXT`：AI 生成下一集
- Market 页生态接入：B.AI 下一集生成、分支预览、封面资产
- 本地生成风格海报资产

下一步最该补：

```text
创作者后台里的 Generate with B.AI 按钮
```

### 一句话理解

```text
B.AI 是 MIRAGEA 的短剧持续生成引擎。
```

## 4. 三者怎么组成闭环

可以这样理解：

```text
B.AI 负责生成内容
$HTX 负责激励和经济循环
HTX API 负责资产和账本同步
```

完整业务链路：

1. B.AI 生成短剧和分支预览。
2. 用户观看短剧。
3. 用户用 $HTX 购买高权重投票或解锁内容。
4. 系统基于投票结果生成预测盘口。
5. 用户用 USDT / HTX / 钱包参与预测。
6. 剧情更新后按投票快照结算。
7. HTX API 同步资产、流水、分润状态。
8. 创作者获得 $HTX 分润。
9. B.AI 继续生成下一集。

## 5. 对比赛最有价值的讲法

如果只讲一个：

```text
$HTX 经济模型
```

因为它直接绑定商业模式。

如果讲技术壁垒：

```text
B.AI 算力驱动生长式短剧生产
```

因为它直接绑定产品差异化。

如果讲生态闭环：

```text
MIRAGEA 使用 B.AI 生成内容，用 $HTX 驱动用户共创和创作者分润，用 HTX API 完成资产与账本同步。
```

## 6. 当前优先级建议

优先级从高到低：

1. `$HTX 经济模型`
2. `B.AI 算力`
3. `HTX API`

原因：

- `$HTX` 最贴近商业模型。
- `B.AI` 最贴近产品差异化。
- `HTX API` 更适合作为后续真实资产底座，比赛 Demo 可以先 mock。
